# 代码审查综合检查清单

审查时按 2a→2b→2c→2d 顺序执行。每项给出 `[P0-P3] file:line — 问题；建议：具体改法`。

---

## 2a. SOLID + 架构

### SRP — 单一职责
- [ ] 类/模块是否处理多个不相关功能？（构造参数 >5、行数 >300、方法 >20）
- [ ] 类中部分方法变更频率明显不同？

### OCP — 开闭原则
- [ ] 频繁 switch/if-elif 基于类型判断而非多态？
- [ ] 新增类型需修改现有函数？

### LSP — 里氏替换
- [ ] 子类是否改变父类方法行为预期？（加强前置条件、减弱后置条件、抛意外异常）

### ISP — 接口隔离
- [ ] 接口方法 >10 个？类被迫空实现不需要的方法？

### DIP — 依赖倒置
- [ ] 高层模块直接 new 低层具体类？难以 mock 测试？

### 代码气味
- [ ] Long Method (>50 行)、Feature Envy、Data Clumps (3+ 字段总是同时出现)
- [ ] Primitive Obsession、Shotgun Surgery、Divergent Change

---

## 2b. 移除候选

- [ ] 未调用的函数/方法？（grep 搜索确认，注意动态调用）
- [ ] 已永久启用的 Feature Flag 旧分支？
- [ ] >10 行注释掉的代码？（git 历史已保留）
- [ ] 未使用的导入/变量？
- [ ] 过时注释（与代码不一致）？TODO/FIXME 超 6 月？
- [ ] 相似度 >80% 的重复代码块？

**删除策略**：无外部调用 + 有测试覆盖 + 无动态调用 → 安全删除；公开 API/跨服务 → 标记 @deprecated 延迟删除。

---

## 2c. 安全扫描

### 注入攻击
- [ ] SQL: 参数化查询 vs 字符串拼接？
- [ ] XSS: 用户输入输出前转义？innerHTML vs textContent？
- [ ] SSRF: 用户提供的 URL 白名单验证？协议限制？
- [ ] 路径穿越: 文件路径含用户输入？os.path.realpath 规范化？

### 认证授权
- [ ] IDOR: 验证当前用户有权访问请求资源？
- [ ] RBAC: 路由/函数级角色检查？
- [ ] Session: 安全随机 ID、合理过期、敏感操作后重新生成？

### JWT
- [ ] 强制指定单一算法（非 none/多算法）？
- [ ] 验证 exp 声明？Payload 不含敏感信息？

### 竞态条件
- [ ] TOCTOU: 检查-使用间资源状态可能变化？用原子操作替代。
- [ ] Check-Then-Act: 检查-执行分离？用数据库原子操作。
- [ ] 共享变量: 锁保护？

### 敏感信息
- [ ] 硬编码密钥/密码/Token？→ 环境变量
- [ ] 日志记录敏感信息（密码/Token/PII）？
- [ ] .gitignore 覆盖 .env/credentials？

### 运行时
- [ ] 网络请求/数据库查询设置超时？
- [ ] CORS 限制允许源（非 `*`）？
- [ ] 状态改变操作验证 CSRF Token？

---

## 2d. 代码质量

### 错误处理
- [ ] 空 catch/吞异常？至少记录日志。
- [ ] 过宽捕获 bare Exception？捕获具体类型。
- [ ] async 函数缺少错误处理？
- [ ] 错误响应泄露 stack trace/内部信息？

### 性能
- [ ] 循环中执行数据库查询 (N+1)？→ eager loading
- [ ] 缓存无 TTL？键无版本？缺少失效策略？
- [ ] 无界集合增长？→ deque(maxlen) / 生成器
- [ ] 热路径正则含回溯风险 (ReDoS)？

### 边界条件
- [ ] truthy check 意外排除 0/空字符串？→ `is not None`
- [ ] 访问属性前未检查 None？
- [ ] 空集合直接 `[0]`/`max()`？→ 检查长度或 default
- [ ] 除零风险？
- [ ] 超长字符串未截断？

### 可维护性
- [ ] 魔法数字？→ 命名常量
- [ ] 3+ 行重复代码？→ 提取函数
- [ ] 函数圈复杂度 >10？

---

## 严重级别速查

| P0 Critical | P1 High | P2 Medium | P3 Low |
|-------------|---------|-----------|--------|
| 注入(SQL/XSS/SSRF) | N+1/无界内存 | 缓存无TTL | 魔法数字 |
| 硬编码密钥 | 逻辑错误 | 空指针风险 | 命名/风格 |
| 数据丢失风险 | 吞异常/信息泄露 | 代码气味 | 注释/文档 |
| 正确性 bug | 明显 SOLID 违反 | 轻微 SOLID 违反 | 可选改进 |

- **P0/P1** → 必须 reject
- **P2** → 建议 reject（影响小可 pass + 记录）
- **仅 P3** → pass + reviewer_notes 记录
