# 文档转换工具 · 使用手册

> 适用版本：markitdown:ocr &nbsp;|&nbsp; 支持离线使用 &nbsp;|&nbsp; 无需联网

---

## 这个工具能做什么？

它能把您电脑里的各类文件——Word 文档、Excel 表格、PPT 演示、PDF、图片——自动转换成纯文字格式（Markdown），方便您：

- 把内容喂给 AI 助手分析、总结
- 提取图片里的中文文字（OCR 识别）
- 把各种格式统一整理成可搜索的文本

**支持的文件格式：**

| 文件类型 | 常见后缀 |
|----------|---------|
| Word 文档 | `.docx` |
| Excel 表格 | `.xlsx`、`.xls`、`.csv` |
| PPT 演示 | `.pptx` |
| PDF 文档 | `.pdf` |
| 图片（含文字识别） | `.png`、`.jpg`、`.jpeg` |
| 网页文件 | `.html` |
| 电子书 | `.epub` |
| 纯文本 | `.txt`、`.json`、`.xml` |

## 第一步：安装前准备

### 安装 Docker

Docker 是运行本工具的"容器环境"，相当于一个独立的小系统，安装一次即可。

1. 打开浏览器，访问 [https://www.docker.com/products/docker-desktop](https://www.docker.com/products/docker-desktop)
2. 点击 **Download for Windows**（Mac 用户选对应版本）
3. 下载完成后双击安装包，按照提示一路点击"下一步"即可
4. 安装完成后，桌面会出现 Docker 图标，双击启动，等待右下角状态变为绿色"Running"

> **提示：** Docker 启动后会常驻后台，以后每次开机会自动启动，无需重复操作。

---

### 加载工具镜像

管理员会提供给您一个镜像文件（名称类似 `markitdown-ocr.tar`），这是工具的完整安装包。

1. 将镜像文件保存到电脑任意位置，例如 `D:\工具\markitdown-ocr.tar`
2. 按 `Win + R`，输入 `cmd`，按回车，打开命令提示符窗口
3. 输入以下命令并按回车（请将路径替换为您实际的文件位置）：

```
docker load -i D:\工具\markitdown-ocr.tar
```

4. 看到 `Loaded image: markitdown:ocr` 说明加载成功

> **只需操作一次**，之后无需重复加载。

---

## 第二步：转换文件

### 基本用法

打开命令提示符窗口（`Win + R` → 输入 `cmd` → 回车），然后按以下格式输入命令：

```
docker run --rm -v "您的文件夹路径:/data" markitdown:ocr /data/文件名 -o /data/输出文件名.md
```

---

### 实际操作示例

**场景一：转换 Word 文档**

假设您有一个文件 `D:\文档\报告.docx`，想转换成文本：

```
docker run --rm -v "D:\文档:/data" markitdown:ocr /data/报告.docx -o /data/报告.md
```

转换完成后，在 `D:\文档\` 文件夹里会出现 `报告.md` 文件。

---

**场景二：提取图片中的文字（OCR）**

假设您有一张截图 `D:\图片\截图.png`，想把图中文字提取出来：

```
docker run --rm -v "D:\图片:/data" markitdown:ocr /data/截图.png -o /data/截图文字.md
```

---

**场景三：转换 Excel 表格**

假设文件路径为 `D:\表格\数据.xlsx`：

```
docker run --rm -v "D:\表格:/data" markitdown:ocr /data/数据.xlsx -o /data/数据.md
```

转换后的表格会以文字表格形式保存。

---

**场景四：转换 PDF**

假设文件路径为 `D:\文档\合同.pdf`：

```
docker run --rm -v "D:\文档:/data" markitdown:ocr /data/合同.pdf -o /data/合同.md
```

---

### 命令结构说明

| 命令片段 | 含义 |
|----------|------|
| `docker run --rm` | 运行容器，完成后自动清理，不占用额外空间 |
| `-v "D:\文档:/data"` | 把您的文件夹（`D:\文档`）挂载到容器里，工具才能读取 |
| `markitdown:ocr` | 使用的工具镜像名称 |
| `/data/报告.docx` | 要转换的文件（`/data/` 后面接文件名） |
| `-o /data/报告.md` | 输出结果保存的文件名（`-o` 表示 output 输出） |

> **关键规则：** `-v` 后面冒号左边是您电脑上的真实路径，冒号右边固定写 `/data`，文件名保持一致。

---

## 第三步：查看结果

转换生成的 `.md` 文件是纯文本格式，可以用以下方式打开：

- **记事本**：右键文件 → 打开方式 → 记事本
- **VS Code**（推荐）：安装后双击即可，支持格式预览
- **直接复制内容**粘贴到 AI 对话框（如 ChatGPT、文心一言等）

## 常见问题

**Q：命令运行后没有反应，窗口一直等待？**

正在处理中，请耐心等待。图片 OCR 识别通常需要 5～30 秒，大文件 PDF 可能更长。出现新的命令行提示符（`>`）即表示完成。

---

**Q：提示"docker: command not found"？**

Docker 未启动或未安装。请检查：
1. 桌面的 Docker 图标是否存在
2. 双击 Docker Desktop 启动，等待右下角图标变绿
3. 重新打开命令提示符窗口后再试

---

**Q：提示"No such file or directory"？**

路径填写有误，请检查：
1. `-v` 后面的文件夹路径是否正确（注意是文件夹路径，不是文件路径）
2. `/data/` 后面的文件名是否和您的实际文件名一致（包括后缀，如 `.docx`、`.png`）
3. 文件名中含有空格时，整个路径需要用英文引号包裹

---

**Q：图片中的文字识别有错误？**

Tesseract OCR 对清晰度高、字体规范的图片识别效果最好。以下情况会影响准确率：
- 图片模糊或分辨率过低
- 字体较特殊（手写体、艺术字）
- 图片倾斜角度较大

建议对重要内容人工核对。

---

**Q：提示"Permission denied"或"权限不足"？**

这是文件夹写入权限问题。请在命令中加上 `--user $(id -u):$(id -g)`，例如：

```
docker run --rm --user $(id -u):$(id -g) -v "/home/用户名/图片:/data" markitdown:ocr /data/图片.png -o /data/结果.md
```

Windows 用户遇到此问题，请联系 IT 支持人员重新构建镜像。

---

**Q：转换结果是乱码？**

用记事本打开时，请选择 **UTF-8 编码**：文件 → 打开 → 右下角编码选"UTF-8"。

## 导出镜像（分发给其他电脑）

如需将工具复制到其他没有网络的电脑：

1. 在已有工具的电脑上，打开命令提示符，输入：

```
docker save markitdown:ocr -o markitdown-ocr.tar
```

2. 将生成的 `markitdown-ocr.tar` 文件（约 1.1 GB）拷贝到 U 盘或共享目录
3. 在目标电脑上按照"**第一步 → 加载工具镜像**"中的步骤操作即可
